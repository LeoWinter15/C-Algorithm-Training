# D. Distributed System — 题解

## 问题回顾

有 $n$ 个工人（编号 $0 \sim n-1$），$q$ 个任务。第 $i$ 个任务有 $a_i$ 个子任务（编号 $0 \sim a_i-1$），子任务 $j$ 分配给工人 $(b_i + j) \bmod n$。求每个工人最终分到多少子任务。

## 核心思路

每个任务相当于：**从工人 $b_i$ 开始，连续 $a_i$ 个工人各得 1 个子任务**（循环）。

把 $a_i$ 拆成两部分：

- **完整轮次**：$\lfloor a_i / n \rfloor$ — 绕一圈算一轮，每个工人都能分到这么多。
- **余数**：$a_i \bmod n$ — 多出来的不够一圈，只有从 $b_i$ 开始的连续 $\text{rem}$ 个工人各多得 1 个。

## 算法步骤

1. 维护一个全局变量 `base`，累加所有任务的 $\lfloor a_i / n \rfloor$（每个工人都有的保底）。
2. 对于余数部分，用**差分数组** `diff` 高效处理循环区间 $+1$。
3. 最后对 `diff` 做前缀和，还原每个工人的"余数额外加成"，再加上 `base` 即得答案。

## 差分数组处理循环区间

对于每个任务的余数 $\text{rem} = a_i \bmod n$，需要在工人 $[b_i,\ b_i + \text{rem} - 1]$（模 $n$）上各 $+1$。

分两种情况：

| 情况 | 区间 | 差分操作 |
|------|------|----------|
| $b + \text{rem} \le n$（不绕回） | $[b,\ b+\text{rem})$ | `diff[b] += 1`，`diff[b+rem] -= 1` |
| $b + \text{rem} > n$（绕回开头） | $[b,\ n-1]$ 和 $[0,\ (b+\text{rem}) \bmod n)$ | `diff[b] += 1`，`diff[0] += 1`，`diff[(b+rem)%n] -= 1` |

## 前缀和还原答案

```cpp
long long cur = 0;
for (int i = 0; i < n; ++i) {
    cur += diff[i];       // 前缀和 = 工人 i 通过余数区间多拿的子任务数
    ans[i] = base + cur;  // 完整轮次 + 余数额外 = 最终结果
}
```

## 样例验证

输入：
```
2
7 3
10 0
4 2
21 1
1 2
200 0
100 0
```

**第一个测试用例**（$n=7, q=3$）：

| 任务 | $a$ | $b$ | 完整轮次 | 余数 | 差分操作 |
|------|-----|-----|----------|------|----------|
| 1 | 10 | 0 | 1 | 3 | `diff[0] += 1, diff[3] -= 1` |
| 2 | 4 | 2 | 0 | 4 | `diff[2] += 1, diff[6] -= 1` |
| 3 | 21 | 1 | 3 | 0 | 无 |

- `base = 1 + 0 + 3 = 4`
- 前缀和：`[1, 1, 2, 1, 1, 1, 0]`
- 答案：`[5, 5, 6, 5, 5, 5, 4]` ✓

**第二个测试用例**（$n=1, q=2$）：

- 两个任务均摊到一个工人，`base = 200 + 100 = 300`，答案 `[300]` ✓

## 复杂度

- **时间**：$O(n + q)$ 每个测试用例，总计 $O(\sum n + \sum q)$
- **空间**：$O(n)$
- 题目保证 $\sum n + \sum q \le 2 \times 10^5$，使用 `long long` 防溢出。
